2. bcrypt kise use krty hain for password encryption and decryption:
A "salt" is a random value added to the password before hashing. It ensures that even if two users have the same password, their hashed values will be different due to the unique salt used for each password.The primary purpose of a salt is to protect against rainbow table attacks, where precomputed hash values are used to crack passwords.
"Salt rounds": refer to the number of times the hashing algorithm is applied during the creation of a hash. In the context of bcrypt, increasing the number of salt rounds makes the hashing process slower, which enhances security.
